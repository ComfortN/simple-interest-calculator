# Contributing Guidelines

Thank you for your interest in contributing to this project.

All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome.

This repository contains a simple interest calculator created as part of a micro-finance startup initiative and an example of migrating projects from Subversion (SVN) to Git.

We welcome contributions from the community.

## How to Contribute

1. Fork the repository.
2. Clone your fork to your local machine.
3. Create a new branch for your changes.
4. Make your improvements or fixes.
5. Commit your changes with a clear commit message.
6. Push the branch to your fork.
7. Open a Pull Request describing the changes you made.

## Reporting Issues

If you find a bug or have a suggestion for improvement, please open an Issue in the repository.
Include as much detail as possible so that maintainers can understand the problem.

## Code Style

Please follow these basic guidelines:

* Keep the code simple and readable.
* Use meaningful variable names.
* Comment code where necessary.
* Test your changes before submitting a Pull Request.

## Pull Request Process

1. Ensure your code works correctly.
2. Update documentation if necessary.
3. Provide a clear explanation of your changes in the Pull Request description.

## Code of Conduct

By participating in this project, you agree to follow the guidelines outlined in the CODE_OF_CONDUCT.md file.
